# Security Policy

## Supported Scope

This repository is a documentation and skill repository. Security reports are most useful when they involve:

- malicious or unsafe automation guidance
- dependency risks in the docs toolchain
- credential leakage in workflow examples or templates

## Reporting

Please do not open a public issue for a suspected security problem.

Instead, email the maintainer or open a private security advisory through GitHub if available.

When reporting, include:

- the affected file or workflow
- reproduction steps
- potential impact
- whether the issue is documentation-only or executable
